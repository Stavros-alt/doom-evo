# Plan to Fix Playtesting Bugs in doom.evo

## Overview
The playtesting revealed several bugs related to unintended carryover of player upgrades and currency between rounds, enemy AI issues (range, roaming, pathfinding), and player survival mechanics. This plan outlines targeted fixes to address each issue based on codebase analysis.

## Issues and Fixes

### 1. Carryover Bugs (Prices, Weapons, Health Upgrades)
**Problem:** Player upgrades (health, speed, armor, weapons) and currency carry over between rounds, activating upon entering round 2. This may make gameplay too easy or unbalanced.

**Root Cause:** In `main.py`, `player_upgrades` dict and `current_money` persist between rounds and are passed to new `GameEngine` instances. Shop purchases accumulate permanently.

**Fix Steps:**
- Modify `main.py` to reset `player_upgrades` and `current_money` at the start of each round (after evolution phase).
- Update `GameEngine` initialization to use base values instead of carried upgrades for each round.
- Clean up duplicate code in `_buy_item` function (duplicate MAX_SPEED and ARMOR blocks).
- Ensure weapons reset to default each round, or clarify if weapon carryover is intended.

**Files to Edit:** `main.py`, `game.py` (GameEngine.__init__).

**Testing:** Verify round 2 starts with base player stats (100 health, 3.5 speed, 0 armor, default weapon).

### 2. Enemy Range Too Low
**Problem:** Enemies have insufficient alert and attack ranges.

**Root Cause:** `alertRadius=12`, `attackRadius=18` in `game.py` enemy creation.

**Fix Steps:**
- Increase `alertRadius` to 20-25 and `attackRadius` to 25-30 in `game.py`.
- Test ranges to ensure enemies detect and engage player from reasonable distances without being too aggressive.

**Files to Edit:** `game.py`.

**Testing:** Enemies should spot and attack player from farther away.

### 3. Enemies Not Roaming Enough
**Problem:** Enemies primarily chase via NN, lacking roaming behavior.

**Root Cause:** Enemy state defaults to chasing; no explicit roaming logic.

**Fix Steps:**
- Add roaming state logic in `_update_enemies`: if no player in sight, switch to patrol mode with random movement.
- Implement patrol waypoints or random direction changes over time.
- Balance roaming vs. chasing to prevent enemies from clustering.

**Files to Edit:** `game.py` (_update_enemies, _move_enemy).

**Testing:** Enemies move around map when player is not visible.

### 4. Poor Pathfinding (Running Into Walls)
**Problem:** Enemies get stuck on walls despite stuck detection.

**Root Cause:** Complex stuck logic with timers and backtracking, but insufficient for all scenarios (comments indicate persistent clipping issues).

**Fix Steps:**
- Simplify stuck detection: reduce thresholds, add more frequent raycasts.
- Improve movement: add smoother turning, better obstacle avoidance.
- Add fallback random movement when stuck for too long.
- Review NN outputs for invalid moves causing oscillation.

**Files to Edit:** `game.py` (_move_enemy method).

**Testing:** Enemies navigate around walls without getting permanently stuck.

### 5. Player Not Dying
**Problem:** Player survives despite enemy attacks.

**Root Cause:** High armor (up to 50% damage reduction) from carryover, or damage not applying correctly.

**Fix Steps:**
- After fixing carryover, verify damage application in `_update_bullets`.
- Reduce armor effectiveness if still too strong (e.g., cap at 40% reduction).
- Ensure bullet collision and damage calculation work properly.
- Add debug logging for damage taken.

**Files to Edit:** `game.py` (_update_bullets).

**Testing:** Player takes damage and can die from enemy attacks.

## Implementation Order
1. Fix carryover bugs (resets and code cleanup).
2. Adjust enemy ranges.
3. Implement enemy roaming.
4. Improve pathfinding.
5. Verify player damage and death mechanics.
